太好了，我们就把**第一阶段（两周）**做成“一次能跑起来、能收集到关键数据的 MVP”。下面是可直接照做的落地方案：目标、功能范围、技术方案、日程表、埋点与验收、上线与拉新、成本与风险控制。你复制这套就能开工。

# 目标与验收

- **目标**：用最小功能验证“跟学/跟练/跟读 + 词书 + 个性化计划雏形”的价值；两周拿到留存与付费意向数据，决定是否开通支付与做浏览器插件。
- **Go/No-Go 指标（两周内）**
   D1≥30%，D7≥12%，**升级页曝光→留联（愿望单）≥2%**，广告投诉率＜0.5%。

# 一阶段功能范围（MVP）

**用户侧**

- 登录/首次引导（1 分钟到首个练习）
- 学习页：逐句播放、倍速、自动下一句、跟读计时、计次
- 生词本：添加/移除/今日复习
- 词书包：内置 2 个（基础核心词、常用短语）
- 计划：**“每日 5 分钟”轻计划**（SRS 预告位），打卡与连续天数
- 导入：文本/URL 粘贴（先适配 1–2 个站点的正文抽取）
- 导出：**长图**（每日 1 次）
- **变现假门**：升级页（去广告/配额翻倍/周报/AI 评分分钟包），**收愿望单**不收款
- 广告：仅结果页底部 Banner + 完成页插屏（频控）
- 意见反馈入口（表单）

**后台/运营**

- 词书与内容管理（简单 JSON/脚本级）
- 订阅消息：每日早/晚提醒（复习/补卡）
- 数据看板：核心埋点查询脚本（留存/曝光/愿望单）

# 技术与架构（极省成本）

**首发全部用微信 CloudBase（云开发）**，不上自建服务器与域名。

- 模块：云函数（Node/Python任选）、云数据库、对象存储、CDN、定时触发器、日志
- 环境变量：`ENV=dev/prod`，`PAY_ENABLED=false`，`ADS_ENABLED=true`，`FEATURE_FLAGS=a,b,c`

**数据集合（建议字段）**

```
users{openid,unionid,nick,avatar,created_at,pro_until,source,last_login}
content{id,type("sentence|dialog|word"),tags,level,text,audio_url,segments[],length,created_at}
progress{user_id,content_id,step("new|learning|review"),streak,last_ts,secs_today}
vocab{user_id,word,phonetic,meaning,srs_level,next_review_at,created_at}
plans{user_id,mode("5min"),quota_items,scheduled_at}
orders{user_id,plan,price,status("intent"),created_at,paid_at,txn_id}
events{user_id,event,props,ts}
wishlists{user_id,plan,price_point,contact("wx|phone|email"),ts}
```

**索引**：`users.openid(unique)`、`progress(user_id,content_id)`、`vocab(user_id,next_review_at)`、`events(user_id,ts)`

**云函数（最小 10 个）**

- `auth.me`（鉴权/拉档）
- `content.list` / `content.get`
- `study.saveProgress`（进度/时长/打卡）
- `vocab.add` / `vocab.list` / `vocab.remove`
- `import.fromUrlOrText`（正文抽取/切句/去噪）
- `export.longshot`（生成长图，返回URL）
- `billing.createIntent`（记录“升级”意向）
- `wishlist.submit`（愿望单）
- `cron.dailyDigest`（定时：明日复习与补卡提醒）

**存储与 CDN**

- 目录：`audio/`、`covers/`、`exports/`；长图 24h 过期；热资源走 CDN

**安全规则**

- `content` 只读；其余按 openid 读写；管理脚本仅限控制台

# 原型流程（关键 5 页）

1. **首登/引导** → 选择词书包 → 进入“今日 5 分钟”
2. **学习页** → 逐句播放/跟读 → 完成后展示进度与“明日复习”
3. **生词本** → 今日复习（SRS）
4. **我的** → 打卡天数/学习时长 → **升级入口（假门）**
5. **升级页**（假门） → 展示权益与价格梯度（A/B：月 9.9 vs 14.9；年 69 vs 99）→ **留联表单**

# 预置内容与文案

- 词书包（各 200–300）：**核心词**、**常用短语**；每条含 `text/meaning/phonetic/audio_url`
- 学习页引导语：3 步→“点播→跟读→下一句”
- 升级页权益文案（简短对照）：去广告 / 配额翻倍 / 周报 / **（预告）AI 评分分钟包**

# 拉新与冷启动（不花钱优先）

- 公众号首文：《7 天口语跟读挑战（含模板卡）》+ 菜单栏直达
- 视频号 3 条 20–40s 实录（“1 分钟看懂跟读节奏/打卡”），评论挂小程序码
- 建体验群（100 人），榜单 + 连续打卡奖励（送“无广告 7 天”**兑换码**）
- 小程序名与描述包含关键词：跟读/口语/打卡/词书

# 成本估算（首月）

- CloudBase：**¥50–200/月**（按量；含 DB/函数/存储/CDN）
- 认证：**0 元**（首阶段不办个体户）
- 域名/服务器：**0 元**（本阶段不需要）
- 广告通道费：无；支付通道费留到二阶段

> 达标后再开个体户（≈¥300/年）与商户号，接微信支付。

# 风险与兜底

- **导入稳定性**：仅支持“文本 + 1–2 个 URL 站点”，失败给友好提示与“复制文本导入”兜底
- **广告干扰**：仅两处，严格频控；收到负反馈即降频或下线
- **合规**：隐私政策与用户协议最简版先上；内容仅供自学，避免分发他人受版权保护的素材

# 二阶段准备（达标即启）

- **支付**：办理个体户+商户号 → 开启订阅（月/年）与**AI 分钟包**
- **插件 MVP（3–5 天）**：**“一键采集器”**（网页/视频→送素材到小程序），先发 Edge 商店
- **AI 评分最小可用**：分钟额度计量 + 逐词热力图（作为强付费锚点）
