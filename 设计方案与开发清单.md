# 详细设计与开发清单（含里程碑）

本文件在《开发计划》基础上，细化到可执行的工程级方案：目录结构、后端模块、前端页面与组件、数据与权限、测试与部署，以及两周的里程碑与 TODO 列表。

## 架构总览
- 前端：微信小程序（组件化、轻量全局状态、埋点统一）。
- 后端：Python FastAPI（SCF HTTP 触发）+ CloudBase 数据库/存储/CDN/定时器。
- 分层：领域模型（domain）/ 服务层（services）/ 仓储层（repositories）/ 基建（infra）。

- 集合：`users, content, progress, vocab, plans, orders, events, wishlists`
- 关键索引：`users.openid(unique)`、`progress(user_id, content_id)`、`vocab(user_id, next_review_at)`、`events(user_id, ts)`

### TCB CommonServiceAPI 请求/响应样例（参考）

请求：新增文档（AddDocument）

```json
{
  "EnvId": "<your-env-id>",
  "Service": "database",
  "Action": "AddDocument",
  "Data": {
    "CollectionName": "vocab",
    "Document": { "word": "hello", "user_id": "openid_123" }
  }
}
```

返回（可能的形态之一）：

```json
{ "Response": { "_id": "abc123" } }
```

请求：查询文档（QueryDocuments）

```json
{
  "EnvId": "<your-env-id>",
  "Service": "database",
  "Action": "QueryDocuments",
  "Data": {
    "CollectionName": "vocab",
    "Filter": { "user_id": "openid_123" },
    "Limit": 10,
    "Offset": 0
  }
}
```

返回（可能的形态之一）：

```json
{
  "Response": {
    "Documents": [ { "_id": "1", "word": "hello", "user_id": "openid_123" } ],
    "Total": 1
  }
}
```

说明：接口版本/字段名在不同环境可能略有差异，`TCBClient` 已做常见 key 的兼容与 `id` 规范化；如与你的返回不同，可在 `TCBClient` 内部微调映射并扩充契约测试。

## 后端模块设计
- 鉴权（`auth.me`）：`wx.login code -> openid` 绑定/拉档；返回 `user{openid,nick,avatar,pro_until}`。
- 内容（`content.list/get`）：分页、分类/难度筛选；只读。
- 学习进度（`study.saveProgress`）：写入 `secs_today, step, streak`；生成事件 `lesson_finish`。
- 生词本（`vocab.add/list/remove`）：SRS 字段 `srs_level, next_review_at`。
- 导入（`import.fromUrlOrText`）：支持 `text|url`；白名单站点正文抽取 + 切句。
- 导出（`export.longshot`）：生成长图，上传 `exports/` 返回 URL。
- 变现假门（`billing.createIntent, wishlist.submit`）：仅记录意向与联系。
- 定时（`cron.dailyDigest`）：订阅消息模板，推送“明日复习/补卡”。
- 横切：日志（`request_id`）、错误码（统一枚举）、埋点 `EventBus.track(event, props, user_id)`。

接口示例（节选）
- `POST /auth/me` body: `{ code }` → `{ user, featureFlags }`
- `GET /content?type=sentence&level=A1&page=1&pageSize=20` → `{ items, total }`
- `POST /study/progress` body: `{ contentId, secs, step }` → `{ ok:true }`
- `POST /vocab` body: `{ word, meaning, phonetic }` → `{ ok:true }`

## 小程序页面与组件
- 页面
  - `index`：首次引导/选词书 → 进入“每日 5 分钟”。
  - `study`：顶部 `player`，中部 `sentence-list`（当前句高亮/自动滚动），底部控制条（倍速/下一句/跟读计时）。
  - `vocab`：今日复习（`review-card`）+ 搜索/新增/移除。
  - `plan`：打卡、连击天数、明日预告。
  - `import`：文本/URL 粘贴，解析进度提示与失败兜底。
  - `export`：长图预览与“保存到相册”。
  - `upgrade`：权益展示 + 愿望单表单（假门）。
  - `mine`：个人信息、学习时长、设置、反馈。
- 组件
  - `player`：倍速、进度、播放/暂停；对外事件 `timeupdate/ended`。
  - `sentence-list`：`props: segments[]; activeIndex`，方法 `scrollTo(index)`。
  - `review-card`：SRS 操作 `again/hard/good/easy`，回调更新。
  - `progress-bar`：完成度与打卡动画。
- 工程
  - 请求封装 `services/request.ts`：`baseURL, timeout, retry, x-openid 透传, 错误弹层`。
  - 全局状态 `store`：`user, plan, todayStats`，使用订阅/发布简化刷新。
  - 埋点 `utils/track.ts`：统一上报 `events` 集合，失败重试 + 批量。

## 配置、部署与开发命令
- 客户端请求默认携带 `x-openid` 头；后端中间件将其注入 `request.state.openid`，写路径均要求该头（`401`）。
- 环境变量：`ENV, PAY_ENABLED, ADS_ENABLED, FEATURE_FLAGS`（仅后端/云函数中配置）。
- 鉴权：`WECHAT_AUTH_ENABLED, WECHAT_APPID, WECHAT_SECRET, WECHAT_MOCK_PREFIX`（启用后优先走微信 `code2session`；失败回退 mock 前缀）
- 仓储切换：`REPO_PROVIDER=memory|tcb`；当为 `tcb` 时需设置 `TCB_ENV_ID/REGION/SECRET_ID/SECRET_KEY/TOKEN`。
- 导入白名单：`IMPORT_WHITELIST=example.com,foo.com`
- 存储与 CDN：
  - `STORAGE_PROVIDER=local|cos`
  - 本地：`LocalStorage` 返回本地路径或 `CDN_BASE/exports/...`
  - COS：配置 `COS_BUCKET, COS_REGION, COS_SECRET_ID, COS_SECRET_KEY, COS_CDN_BASE(可选)`；上传后返回 COS/CDN URL
- 本地：`make venv && source .venv/bin/activate && make deps && make dev`（`uvicorn app.main:app --reload`）。
- 测试：`make test`（`pytest -q`）；覆盖率门槛：写路径 ≥80%。
- 部署：`make package && make deploy`（SCF），`cloudbase functions:deploy --only <name>`，静态托管 `cloudbase hosting:deploy`。

### 导入与导出（开发阶段）
- 导入
  - 文本：`POST /import {type:"text", payload:"..."}` → 切句入库返回 `segments`
  - URL：`POST /import {type:"url", payload:"https://example.com/x/y"}` → 基于域名白名单（`IMPORT_WHITELIST`）做最小占位解析；生产环境建议对接正文抽取
- 导出
  - `POST /export/longshot {contentId}` → 使用 Pillow 生成长图，开发环境输出到 `exports/longshot_<id>.png` 并返回路径（生产建议上传到对象存储并返回 CDN URL）

- 合规与广告：收到负反馈即时降频或下线；敏感文案走白名单。
