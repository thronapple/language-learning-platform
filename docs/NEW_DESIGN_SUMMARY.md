### 1. [水平评估系统设计](./ASSESSMENT_DESIGN.md)
**核心内容**:

**差异化价值**:
- ❌ 通用词包: "已背500词"

### 5. [用户旅程设计](./USER_JOURNEY_DESIGN.md)

**交互亮点**:
- 倒计时脉动动画(紧迫感)
- 就绪度环形图(进度可视化)
- 庆祝动画(正向反馈)
- 雷达图(能力可视化)

## 🔧 技术架构

### 技术栈
```
前端: 微信小程序 + React(Web)
后端: Python FastAPI
数据库: CloudBase(MongoDB兼容)
缓存: Redis
存储: CloudBase对象存储/COS
CDN: 腾讯云CDN
```

### 核心算法
1. **自适应评估** - IRT模型(简化版)
2. **场景优先级** - 多因素加权(紧迫度+匹配度+难度)
3. **SRS词汇复习** - SuperMemo SM-2
4. **同步冲突解决** - Last-Write-Wins

### 性能目标
- 首屏加载 < 1.5秒
- 接口响应 < 200ms
- 同步延迟 < 5秒
- 动画帧率 = 60fps

## 🤔 决策记录

### 为什么选择场景化?
**问题**: 通用词包(欧路/Anki)已成熟,难以差异化竞争

**分析**:
- 用户真实需求: "2周后德国出差"
- 学习动机: 不是"背1000词",是"机场/会议不尴尬"
- 进度呈现: "机场85%就绪"比"已背500词"更有紧迫感

**决策**: 以场景为核心,构建差异化优势

---

### 为什么要评估系统?
**问题**: 如何避免"从零开始"或"太简单"?

**分析**:
- 用户水平差异大(A1-C2)
- 通用计划无法满足个性化需求
- 评估可精准定位薄弱环节

**决策**: 首次使用必须评估,生成定制计划

---

### 为什么要多端同步?
**问题**: 小程序是否足够?

**分析**:
- 碎片化学习: 手机背单词 → 电脑复习
- 数据连续性: 学习进度不中断
- 竞争力提升: 避免数据孤岛

**决策**: MVP小程序验证,Phase 2加入Web端
